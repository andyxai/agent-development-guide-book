## 31.1 量化技术（Quantization）

**总**：量化将 FP16/BF16 权重转换为 INT8/INT4，显存占用降至 1/2 或 1/4，是实现本地部署的关键技术。

### 1. 量化原理

**核心思想**：用低精度整数替代高精度浮点数，牺牲少量精度换取显著显存节省。

```
FP16: 16 bits (2 bytes) per parameter
INT8:  8 bits (1 byte) per parameter  → 显存节省 50%
INT4:  4 bits (0.5 bytes) per parameter → 显存节省 75%
```

**7B 模型显存对比**：
- FP16: ~14GB
- INT8: ~7GB
- INT4: ~3.5GB

### 2. 主流量化方法对比

| 方法 | 原理 | 显存 (7B) | 精度损失 | 推理速度 | 适用场景 |
|------|------|-----------|----------|----------|---------|
| **FP16** | 基线 | ~14GB | 0% | 1x | 基线对比 |
| **INT8 (BNB)** | 动态范围缩放 | ~7GB | 1-2% | 1.5-2x | 通用推理 |
| **INT4 (AWQ)** | 激活值保护重要权重 | ~3.5GB | 2-4% | 3-4x | 端侧部署 |
| **INT4 (GPTQ)** | Hessian 二阶信息补偿 | ~3.5GB | 3-5% | 3-4x | 高吞吐服务 |
| **NF4 (QLoRA)** | NormalFloat4 + 双重量化 | ~3.5GB | 2-3% | 2-3x | 微调训练 |

### 3. AWQ（Activation-aware Weight Quantization）

**核心创新**：基于激活值保护重要权重，INT4 精度损失最小。

**关键发现**：
- 并非所有权重都同等重要
- 少量权重对激活值影响巨大（salient weights）
- 保护这些权重，量化其他权重

### 4. 实现示例

#### BitsAndBytes 4-bit 量化

```python
from transformers import AutoModelForCausalLM, BitsAndBytesConfig
import torch

# BitsAndBytes 4-bit 量化配置
bnb_config = BitsAndBytesConfig(
    load_in_4bit=True,
    bnb_4bit_quant_type="nf4",           # NormalFloat4
    bnb_4bit_compute_dtype=torch.bfloat16,
    bnb_4bit_use_double_quant=True,      # 双重量化
)

model = AutoModelForCausalLM.from_pretrained(
    "meta-llama/Llama-2-7b-hf",
    quantization_config=bnb_config,
    device_map="auto",
)
```

#### GPTQ 量化模型加载

```python
from transformers import GPTQConfig

gptq_config = GPTQConfig(
    bits=4,
    group_size=128,
    dataset="c4",
    desc_act=False,
)

model = AutoModelForCausalLM.from_pretrained(
    "TheBloke/Llama-2-7B-GPTQ",
    device_map="auto",
    torch_dtype=torch.float16,
)
```

### 5. 最佳实践参数

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| 量化方法 | AWQ INT4 | 精度最高 |
| 校准数据集 | 128-512 样本 | 代表性数据 |
| Group size | 128 (默认) | 64 更高精度但更慢 |
| 双重量化 | True | 额外节省 30% 显存 |

**总**：推理部署优先选择 AWQ INT4（精度最高），微调使用 QLoRA NF4（显存最低）。

## 31.2 知识蒸馏（Knowledge Distillation）

**总**：知识蒸馏让小模型学习大模型的行为，以较低成本获得接近大模型的性能。

### 1. 蒸馏原理

**核心思想**：训练小模型（Student）模仿大模型（Teacher）的输出。

**三种蒸馏方法**：

| 方法 | 蒸馏目标 | 效果 | 显存开销 |
|------|---------|------|----------|
| **Logits 蒸馏** | 模仿 softmax 输出 | 70-80% 教师性能 | 低 |
| **Hidden States 蒸馏** | 对齐中间层特征 | 80-85% 教师性能 | 中 |
| **Reverse KLD** | 避免 overconfidence | 85-90% 教师性能 | 中 |

### 2. Logits 蒸馏实现

```python
import torch
import torch.nn.functional as F

def kd_loss(student_logits, teacher_logits, temperature=2.0):
    """Knowledge Distillation Loss"""
    student_probs = F.log_softmax(student_logits / temperature, dim=-1)
    teacher_probs = F.softmax(teacher_logits / temperature, dim=-1)
    loss = F.kl_div(student_probs, teacher_probs, reduction="batchmean")
    return loss * (temperature ** 2)
```

### 3. Hidden States 蒸馏

```python
class HiddenStateDistillation:
    def __init__(self, teacher_dim=4096, student_dim=2048):
        # 投影层对齐维度
        self.projection = torch.nn.Linear(student_dim, teacher_dim)
    
    def compute_loss(self, student_hidden, teacher_hidden):
        # student_hidden: (batch, seq_len, student_dim)
        # teacher_hidden: (batch, seq_len, teacher_dim)
        student_proj = self.projection(student_hidden)
        loss = F.mse_loss(student_proj, teacher_hidden)
        return loss
```

### 4. 完整训练循环

```python
temperature = 2.0
alpha = 0.5  # 蒸馏损失权重

for batch in dataloader:
    # Teacher 前向（无梯度）
    with torch.no_grad():
        teacher_output = teacher_model(**batch)
        teacher_logits = teacher_output.logits
        teacher_hidden = teacher_output.hidden_states[-1]
    
    # Student 前向
    student_output = student_model(**batch, output_hidden_states=True)
    student_logits = student_output.logits
    student_hidden = student_output.hidden_states[-1]
    
    # 组合损失
    kd_logits = kd_loss(student_logits, teacher_logits, temperature)
    kd_hidden = hidden_distill.compute_loss(student_hidden, teacher_hidden)
    loss = alpha * kd_logits + (1 - alpha) * kd_hidden
    
    loss.backward()
    optimizer.step()
```

### 5. 关键技术要点

1. **温度系数**：2.0（通用），5.0（复杂任务）
2. **目标模块**：attention + FFN 层
3. **数据量**：10K-100K 样本
4. **学习率**：2e-5 ~ 5e-5
5. **TinyLlama 案例**：用 Llama-2-7B 蒸馏，1.1B 参数达到 7B 的 75% 性能

### 6. 最佳实践参数

| 方法 | 温度系数 | 损失权重 | 效果 | 显存开销 |
|------|----------|----------|------|----------|
| **Logits KD** | 2.0-5.0 | 0.3-0.7 | 70-80% 教师性能 | 低 |
| **Hidden States KD** | - | 0.5-0.8 | 80-85% 教师性能 | 中 |
| **Reverse KLD** | 1.0-2.0 | 0.5 | 85-90% 教师性能 | 中 |
| **Data Augmentation** | - | - | 75-95% 教师性能 | 低 |

**总**：知识蒸馏是训练端侧小模型的有效方法，TinyLlama 1.1B 通过蒸馏可达到 7B 模型的 75% 性能。

## 31.3 LoRA 微调

**总**：LoRA 通过低秩分解实现参数高效微调，训练参数仅 0.5%，显存需求降低 75-95%。

### 1. LoRA 原理

**核心思想**：冻结预训练权重，注入可训练的低秩分解矩阵。

```
ΔW = B × A
```

其中：
- W 是原始权重矩阵（d×k）
- B 是 d×r 矩阵
- A 是 r×k 矩阵
- r << d, k（通常 r=8-64）

### 2. QLoRA 实现示例

```python
from peft import LoraConfig, get_peft_model, prepare_model_for_kbit_training
from transformers import AutoModelForCausalLM, BitsAndBytesConfig
from trl import SFTTrainer
import torch

# 1. QLoRA 4-bit 量化配置
bnb_config = BitsAndBytesConfig(
    load_in_4bit=True,
    bnb_4bit_quant_type="nf4",
    bnb_4bit_compute_dtype=torch.bfloat16,
    bnb_4bit_use_double_quant=True,
)

# 2. 加载模型
model = AutoModelForCausalLM.from_pretrained(
    "meta-llama/Llama-2-7b-hf",
    quantization_config=bnb_config,
    device_map="auto",
)
model = prepare_model_for_kbit_training(model)

# 3. LoRA 配置
lora_config = LoraConfig(
    r=16,                              # Rank
    lora_alpha=32,                     # Alpha (2x rank)
    lora_dropout=0.05,                 # Dropout
    target_modules=[                   # 目标模块
        "q_proj", "k_proj", "v_proj", "o_proj",
        "gate_proj", "up_proj", "down_proj",
    ],
    bias="none",
    task_type="CAUSAL_LM",
)

# 4. 创建 PEFT 模型
peft_model = get_peft_model(model, lora_config)
peft_model.print_trainable_parameters()
# 输出: trainable params: 33M || all params: 6.7B || trainable%: 0.49%

# 5. 训练配置
training_args = TrainingArguments(
    output_dir="./qlora-output",
    num_train_epochs=3,
    per_device_train_batch_size=4,
    gradient_accumulation_steps=4,
    learning_rate=2e-4,                # LoRA 学习率
    weight_decay=0.01,
    warmup_ratio=0.03,
    lr_scheduler_type="cosine",
    optim="paged_adamw_8bit",          # Paged Optimizer
    gradient_checkpointing=True,
    bf16=True,
)

# 6. 训练
trainer = SFTTrainer(
    model=peft_model,
    args=training_args,
    train_dataset=train_dataset,
    tokenizer=tokenizer,
    max_seq_length=2048,
)
trainer.train()

# 7. 保存适配器（仅 30-100MB）
peft_model.save_pretrained("./my-lora-adapter")
```

### 3. 超参数选择指南

| 任务类型 | Rank | Alpha | Dropout | 学习率 |
|----------|------|-------|---------|--------|
| **简单分类** | 4-8 | 8-16 | 0.05 | 1e-4 |
| **指令微调** | 16-32 | 32-64 | 0.05-0.1 | 2e-4 |
| **领域适配** | 64-128 | 128 | 0.1 | 1e-4 |

### 4. 显存需求对比

| 模型规模 | 全量微调 | LoRA (FP16) | QLoRA (NF4) | Adapter 大小 |
|----------|----------|-------------|-------------|-------------|
| **7B** | ~120GB | ~28GB | ~6GB | 30-100MB |
| **13B** | ~220GB | ~52GB | ~10GB | 60-200MB |
| **70B** | ~1.2TB | ~280GB | ~48GB | 300-500MB |

### 5. 关键技术要点

1. **Rank 选择**：从 16 开始，性能不足再增加
2. **Alpha 设置**：固定为 2×rank
3. **Target Modules**：全模块（7 个 proj）比仅 attention 多 3-5% 性能
4. **学习率**：2e-4（通用），避免 >5e-4 导致不稳定
5. **Batch Size**：实际 batch = per_device × gradient_accumulation ≥ 32
6. **Epochs**：2-3 轮，避免 overfitting

**总**：QLoRA 让 65B 模型在单卡 48GB 上可训练（QLoRA 论文原版），70B 模型约需 46GB 显存（单卡 48GB 勉强可行，需严格优化），adapter 仅 300-500MB，是参数高效微调的首选方法。

## 31.4 vLLM 推理加速

**总**：vLLM 通过 PagedAttention 和 Continuous Batching 技术，相比未优化的 HuggingFace Transformers 吞吐量提升最高 24 倍，相比其他优化框架提升约 3.7 倍。

### 1. 核心技术

#### PagedAttention

**问题**：传统推理中 60-80% 显存被 KV Cache 碎片浪费。

**解决方案**：
- 将 KV Cache 分页管理（类似操作系统虚拟内存）
- 非连续块存储，消除碎片
- 显存利用率提升至 90-95%

#### Continuous Batching

**问题**：静态批处理等待整个批次完成，GPU 利用率低。

**解决方案**：
- 请求完成立即插入新请求
- 动态批处理，GPU 持续工作
- 吞吐量提升 2-4 倍

### 2. 性能对比

| 框架 | 吞吐量 (tokens/s) | 并发能力 | 显存利用 |
|------|-------------------|---------|---------|
| **HuggingFace** | ~41 | 低 | 40-60% |
| **TGI** | ~200 | 中 | 60-70% |
| **vLLM** | ~793 | 高 | 90-95% |

**vLLM 优势**：吞吐量是 HuggingFace 的 **19 倍**，TGI 的 **4 倍**。

### 3. 部署示例

#### 启动推理服务

```bash
python -m vllm.entrypoints.openai.api_server \
    --model meta-llama/Llama-2-7b-hf \
    --tensor-parallel-size 1 \
    --max-model-len 4096 \
    --gpu-memory-utilization 0.9 \
    --enable-prefix-caching \
    --disable-log-requests
```

#### 量化模型部署

```bash
python -m vllm.entrypoints.openai.api_server \
    --model TheBloke/Llama-2-7B-AWQ \
    --quantization awq \
    --max-num-batched-tokens 8192 \
    --max-num-seqs 256
```

#### Python API 使用

```python
from vllm import LLM, SamplingParams

# 初始化模型
llm = LLM(
    model="meta-llama/Llama-2-7b-hf",
    tensor_parallel_size=1,
    gpu_memory_utilization=0.9,
    max_model_len=4096,
    enable_prefix_caching=True,  # 启用前缀缓存
)

# 采样参数
sampling_params = SamplingParams(
    temperature=0.7,
    top_p=0.9,
    max_tokens=100,
)

# 批量推理（Continuous Batching 自动优化）
prompts = [
    "Hello, my name is",
    "The president of the United States is",
]
outputs = llm.generate(prompts, sampling_params)

for output in outputs:
    print(f"Prompt: {output.prompt!r}, Generated: {output.outputs[0].text}")
```

### 5. 最佳实践参数

| 配置项 | 推荐值 | 说明 |
|--------|--------|------|
| **gpu_memory_utilization** | 0.9-0.95 | 预留 5-10% 给系统 |
| **max_model_len** | 4096-8192 | 根据业务需求 |
| **max_num_seqs** | 256-512 | 最大并发请求数 |
| **tensor_parallel_size** | GPU 数量 | 多卡并行 |
| **enable_prefix_caching** | True | 多轮对话必开 |

### 6. 优化技巧

1. **KV Cache 优化**：`--gpu-memory-utilization 0.95` 最大化利用
2. **批量大小**：`--max-num-batched-tokens 8192` 控制内存
3. **量化加速**：AWQ/GPTQ INT4 提升吞吐 3-4x
4. **Prefix Caching**：多轮对话/相似 prompt 场景加速 5-10x
5. **Stream 响应**：降低 TTFT（首 token 延迟）
6. **多实例负载均衡**：Nginx/HAProxy 分发请求

**总**：vLLM 是生产环境部署 LLM 的首选引擎，PagedAttention + Continuous Batching 实现极致吞吐量。

## 31.5 简单举例

某创业公司需要在单卡 RTX 4090 (24GB) 上部署 7B 模型：

**方案 1：直接部署**
- FP16: 需要 14GB 显存 ✅ 可行
- 吞吐量：~200 tokens/s

**方案 2：量化部署**
- AWQ INT4: 仅需 3.5GB 显存 ✅ 可行
- 吞吐量：~600 tokens/s（提升 3 倍）
- 精度损失：2-4%

**方案 3：QLoRA 领域微调 + 量化部署**
1. 使用 QLoRA 在领域数据上微调 3 轮（显存 6GB）
2. 保存 adapter（50MB）
3. AWQ INT4 量化
4. vLLM 部署生产服务

**最终效果**：
- 显存占用：3.5GB（原 14GB，节省 75%）
- 吞吐量：600 tokens/s（原 200，提升 3 倍）
- 领域性能：提升 15-20%（QLoRA 微调）
- 并发能力：256 请求（Continuous Batching）
