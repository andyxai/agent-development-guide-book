## 20.1 微调 vs Prompt Engineering

### 20.1.1 何时需要微调？

**决策框架**:

```
需要定制化吗？
    │
    ├─ 否 → 直接用基础模型
    │
    └─ 是 → 任务复杂度？
            │
            ├─ 低（简单格式） → Prompt Engineering
            │
            └─ 高（风格/领域知识） → 数据量？
                    │
                    ├─ 少（<100 条） → Prompt Engineering + Few-shot
                    │
                    └─ 多（>100 条） → 微调
```

**对比表**:

| 维度 | Prompt Engineering | 微调 |
|------|-------------------|------|
| **适用场景** | 简单任务、快速验证 | 风格定制、领域知识 |
| **数据需求** | 无需训练数据 | 100-10000+ 条样本 |
| **成本** | 低（$0.01-0.1/次） | 高（$100-10000+ 一次性） |
| **效果持久性** | 临时（每次都要写 Prompt） | 持久（模型本身改变） |
| **延迟** | 低（无需额外推理） | 中（微调模型推理） |
| **维护成本** | 高（Prompt 版本管理） | 低（模型部署后稳定） |

**漫剧项目决策**:
- **正文生成**: 微调（文风一致性要求高，有 1000+ 条样本）
- **设定检索**: Prompt Engineering（RAG 已足够）
- **大纲规划**: Prompt Engineering（任务结构化，无需微调）
- **质量审核**: Prompt Engineering（规则明确，无需微调）

### 20.1.2 微调方法对比

| 方法 | 原理 | 训练参数 | 显存需求 | 训练时间 | 效果 | 适用场景 |
|------|------|---------|---------|---------|------|---------|
| **全量微调** | 更新所有参数 | 7B 参数 | 80GB+ | 10-50 小时 | 最佳 | 资源充足、追求极致效果 |
| **LoRA** | 更新低秩矩阵 | 0.1% 参数 | 24GB+ | 1-10 小时 | 接近全量 | **推荐**，性价比最高 |
| **QLoRA** | LoRA + 4bit 量化 | 0.1% 参数 | 16GB+ | 1-10 小时 | 略低于 LoRA | 显存受限场景 |
| **P-Tuning** | 更新 Prompt 嵌入 | 0.01% 参数 | 16GB+ | 0.5-5 小时 | 中等 | 快速验证 |

**推荐**: **LoRA** 是性价比最高的选择，效果接近全量微调，训练成本降低 90%+。

```python
import re

def clean_text(text):
    """清洗文本数据"""
    # 移除特殊字符
    text = re.sub(r'[^\w\s\u4e00-\u9fff,.!?;:，。！？；：]', '', text)
    # 移除多余空格
    text = re.sub(r'\s+', ' ', text)
    # 统一标点
    text = text.replace('?', '?').replace('?', '?')
    return text.strip()

# 应用清洗
for sample in training_data:
    sample['input'] = clean_text(sample['input'])
    sample['output'] = clean_text(sample['output'])
```

### 20.2.2 阶段 2：训练配置

**LoRA 训练配置**（使用 HuggingFace + PEFT）:

```python
from transformers import AutoModelForCausalLM, AutoTokenizer, TrainingArguments
from peft import LoraConfig, get_peft_model, TaskType
import torch

# 1. 加载基础模型
model_name = "Qwen/Qwen-7B-Chat"  # 通义千问 7B 对话版
tokenizer = AutoTokenizer.from_pretrained(model_name, trust_remote_code=True)
tokenizer.pad_token = tokenizer.eos_token

model = AutoModelForCausalLM.from_pretrained(
    model_name,
    torch_dtype=torch.float16,  # 半精度训练，节省显存
    device_map="auto",  # 自动分配设备
    trust_remote_code=True
)

# 2. 配置 LoRA
lora_config = LoraConfig(
    task_type=TaskType.CAUSAL_LM,
    r=8,  # 低秩矩阵的秩（8-32，越大效果越好但显存需求越高）
    lora_alpha=32,  # LoRA 缩放因子（通常是 r 的 2-4 倍）
    lora_dropout=0.1,  # Dropout 率（防止过拟合）
    target_modules=["q_proj", "v_proj"],  # 目标模块（Attention 的 Q/V 矩阵）
    bias="none",
    inference_mode=False
)

# 3. 应用 LoRA
model = get_peft_model(model, lora_config)
model.print_trainable_parameters()

# 输出:
# trainable params: 4,194,304 || all params: 7,000,000,000 || trainable%: 0.06%
# 只训练 0.06% 的参数！

# 4. 训练参数配置
training_args = TrainingArguments(
    output_dir="./manhua_lora_model",
    num_train_epochs=3,  # 训练轮数
    per_device_train_batch_size=4,  # 每设备批次大小
    gradient_accumulation_steps=4,  # 梯度累积步数（等效于 batch_size=16）
    learning_rate=2e-4,  # 学习率
    weight_decay=0.01,  # 权重衰减（防止过拟合）
    warmup_ratio=0.1,  # 预热比例
    lr_scheduler_type="cosine",  # 学习率调度器
    logging_steps=10,  # 日志步数
    save_strategy="epoch",  # 保存策略（每轮保存）
    evaluation_strategy="epoch",  # 评估策略
    fp16=True,  # 混合精度训练
    gradient_checkpointing=True,  # 梯度检查点（节省显存）
    report_to="tensorboard",  # 日志工具
)
```

### 20.2.3 阶段 3：训练执行

**数据加载与训练**:

```python
from transformers import DataCollatorForLanguageModeling
from datasets import Dataset
from transformers import Trainer

# 1. 准备数据集
def format_example(example):
    """格式化样本"""
    text = f"### Instruction:\n{example['instruction']}\n\n### Input:\n{example['input']}\n\n### Response:\n{example['output']}"
    return text

train_texts = [format_example(d) for d in train_data]
eval_texts = [format_example(d) for d in eval_data]

# Token 化
train_encodings = tokenizer(train_texts, truncation=True, padding=True, max_length=512)
eval_encodings = tokenizer(eval_texts, truncation=True, padding=True, max_length=512)

# 转换为 Dataset
train_dataset = Dataset.from_dict(train_encodings)
eval_dataset = Dataset.from_dict(eval_encodings)

# 2. 数据整理器
data_collator = DataCollatorForLanguageModeling(
    tokenizer=tokenizer,
    mlm=False  # 因果语言模型，不用 MLM
)

# 3. 创建 Trainer
trainer = Trainer(
    model=model,
    args=training_args,
    train_dataset=train_dataset,
    eval_dataset=eval_dataset,
    data_collator=data_collator,
)

# 4. 开始训练
print("开始训练...")
trainer.train()

# 5. 保存模型
trainer.save_model("./manhua_lora_model")
tokenizer.save_pretrained("./manhua_lora_model")

print("训练完成！模型已保存到 ./manhua_lora_model")
```

### 20.2.4 阶段 4：模型评估

**自动评估指标**:

```python
from rouge import Rouge
from nltk.translate.bleu_score import sentence_bleu

# 1. 加载验证集
test_samples = eval_data[:20]  # 取 20 个样本

# 2. 生成预测
predictions = []
references = []

for sample in test_samples:
    input_text = format_example(sample)
    inputs = tokenizer(input_text, return_tensors="pt").to(model.device)
    
    # 生成
    outputs = model.generate(
        **inputs,
        max_new_tokens=256,
        temperature=0.7,
        top_p=0.9
    )
    
    prediction = tokenizer.decode(outputs[0], skip_special_tokens=True)
    predictions.append(prediction)
    references.append(sample['output'])

# 3. 计算 ROUGE 分数
rouge = Rouge()
rouge_scores = rouge.get_scores(predictions, references)
avg_rouge = {
    'rouge-1': sum(s['rouge-1']['f'] for s in rouge_scores) / len(rouge_scores),
    'rouge-2': sum(s['rouge-2']['f'] for s in rouge_scores) / len(rouge_scores),
    'rouge-l': sum(s['rouge-l']['f'] for s in rouge_scores) / len(rouge_scores),
}

print(f"ROUGE-1: {avg_rouge['rouge-1']:.3f}")
print(f"ROUGE-2: {avg_rouge['rouge-2']:.3f}")
print(f"ROUGE-L: {avg_rouge['rouge-l']:.3f}")

# 4. 计算 BLEU 分数（需要分词）
import jieba

bleu_scores = []
for pred, ref in zip(predictions, references):
    pred_words = list(jieba.cut(pred))
    ref_words = [list(jieba.cut(ref))]
    bleu = sentence_bleu(ref_words, pred_words)
    bleu_scores.append(bleu)

print(f"BLEU: {sum(bleu_scores) / len(bleu_scores):.3f}")
```

### 20.2.5 阶段 5：模型部署

**部署为 API 服务**:

```python
from fastapi import FastAPI
from pydantic import BaseModel
import torch

app = FastAPI()

# 加载微调后的模型
model_path = "./manhua_lora_model"
tokenizer = AutoTokenizer.from_pretrained(model_path, trust_remote_code=True)
model = AutoModelForCausalLM.from_pretrained(
    model_path,
    torch_dtype=torch.float16,
    device_map="auto",
    trust_remote_code=True
)

class GenerateRequest(BaseModel):
    outline: str
    max_length: int = 512

class GenerateResponse(BaseModel):
    content: str
    model_version: str

@app.post("/generate", response_model=GenerateResponse)
async def generate(request: GenerateRequest):
    """生成漫剧正文"""
    input_text = f"### Instruction:\n根据漫剧大纲生成正文\n\n### Input:\n{request.outline}\n\n### Response:\n"
    
    inputs = tokenizer(input_text, return_tensors="pt").to(model.device)
    outputs = model.generate(
        **inputs,
        max_new_tokens=request.max_length,
        temperature=0.7,
        top_p=0.9
    )
    
    content = tokenizer.decode(outputs[0], skip_special_tokens=True)
    
    return GenerateResponse(
        content=content,
        model_version="manhua-lora-v1.0"
    )
```

## 20.4 最佳实践与陷阱

### 20.4.1 最佳实践

1. **数据质量 > 数据数量**: 100 条高质量样本 > 1000 条低质量样本
2. **从小模型开始**: 先用 7B 模型验证流程，再考虑大模型
3. **LoRA 是首选**: 性价比最高，效果接近全量微调
4. **早停防止过拟合**: 监控验证损失，如上升则提前停止
5. **多轮迭代**: 第一轮微调后，根据效果补充数据，再微调

### 20.4.2 常见陷阱

| 陷阱 | 表现 | 解决方案 |
|------|------|---------|
| **数据泄露** | 验证集混入训练集 | 严格划分训练/验证集 |
| **过拟合** | 训练损失低，验证损失高 | 早停、增加 Dropout、减少训练轮数 |
| **欠拟合** | 训练损失和验证损失都高 | 增加训练轮数、增加模型容量 |
| **灾难性遗忘** | 微调后通用能力下降 | 混合通用数据、使用 LoRA |
| **显存溢出** | 训练中途 OOM | 减少批次大小、梯度累积、梯度检查点 |
